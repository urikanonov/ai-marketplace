# commentable-html feature specification

commentable-html turns a standalone HTML artifact into a self-contained code-review surface. A reader can select text, code, diff lines, mermaid nodes or whole diagrams, headings, images, and charts, leave comments in the page, copy a machine-readable review bundle back to an agent, and let the agent mark processed comments handled in the same HTML file.

The core invariant is self-contained operation. Standalone mode embeds the CSS, state blocks, UI, and runtime in one file. NonPortable mode keeps the document-owned state inline but loads CSS and JS from local companion files in `dist/`; its export action rebuilds one portable file so the review can travel without companions. The review layer is bundled into the document. Optional host features, such as mermaid rendering, Chart.js, tooltip libraries, and Run in Azure Data Explorer links, can load from a CDN and should be vendored, inlined, or captured with Export Offline when single-file zero-network portability is required.

Current UI label note: the authoritative action label is `Export as Portable`. Some historical test names still say `Save comments`, `Save in HTML`, or `Export with embedded comments`.

Coverage notation: each row lists the strongest automated coverage or source citation I found. `manual/validator` means the behavior is a documented convention or a static validator invariant rather than a browser E2E behavior test.
