## Packaging and first-run extraction

The plugin ships a compact `skills/commentable-html/` (only `SKILL.md`, `LICENSE`, and a single `skill-resources.zip`) so the CLI installer writes very few files. This matters on Windows, where Defender briefly locks each file as it is written and the installer aborts the whole install on the first transient lock (the `Access is denied. (os error 5)` failure) - a large file count almost never gets through. The bulky runtime (`tools/`, `references/`, `dist/`, `vendor/`) is assembled by `build.py` into `skill-resources.zip`, and a SessionStart hook unpacks it on first run per version, retrying the same transient lock the installer does not. The large tutorial (`docs/`) and worked examples (`examples/`) are NOT shipped; they live at the plugin top level and online. The editable + built skill STAGE tree lives at `dev/skill/`.
