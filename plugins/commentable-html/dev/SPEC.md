# commentable-html feature specification

commentable-html turns a standalone HTML artifact into a self-contained code-review surface. A reader can select text, code, diff lines, mermaid nodes or whole diagrams, headings, images, and charts, leave comments in the page, copy a machine-readable review bundle back to an agent, and let the agent mark processed comments handled in the same HTML file.

The core invariant is self-contained operation. Standalone mode embeds the CSS, state blocks, UI, and runtime in one file. NonPortable mode keeps the document-owned state inline but loads CSS and JS from local companion files in `dist/`; its export action rebuilds one portable file so the review can travel without companions. The review layer is bundled into the document. Optional host features, such as mermaid rendering, Chart.js, tooltip libraries, and Run in Azure Data Explorer links, can load from a CDN and should be vendored or inlined when single-file portability is required.

Current UI label note: the authoritative action label is `Export as Portable`. Some historical test names still say `Save comments`, `Save in HTML`, or `Export with embedded comments`.

Coverage notation: each row lists the strongest automated coverage or source citation I found. `manual/validator` means the behavior is a documented convention or a static validator invariant rather than a browser E2E behavior test.

## Coverage gaps

The following features are documented but lack full automated behavior coverage, or have a known mismatch between spec and documentation:

**Remaining manual or design-only gaps:**
- CMH-CONTENT-05: ADO resource links in authored content are a manual authoring convention with no validator or E2E enforcement.
- CMH-CONTENT-06: prose cross-reference links are only partially covered by TOC anchor tests.
- CMH-KQL-01: the convention that every KQL block has a Run in Azure Data Explorer link is not enforced for arbitrary KQL blocks. Tests cover the demo block, helper output, and validator warnings for links that exist.
- CMH-CHART-05: chart recipe semantics, dark chart palette choices, and data hygiene rules are documentation and review conventions, not automated tests.
- CMH-NET-01 / CMH-MMDLOAD-01: CDN opt-in and local vendoring choices are manual. CMH-MMDLOAD-01 now has E2E coverage for the default CDN path; manual guidance for vendor/self-host advice remains.
- CMH-PRINT-01: intentional non-feature; no test required.

**Documentation/implementation mismatches (spec is correct; docs or tooling do not reflect it):**
- CMH-COPY-06: `references/limitations.md` implies all copy paths have the `prompt` fallback; only Copy all does. The per-block no-prompt rule now has E2E coverage, but the docs still need alignment outside this tests-only pass.
- CMH-SEC-01: the "callers must sanitize untrusted host HTML before wrapping" rule is stated in the spec but has no enforcement in the validator or tooling and is absent from `SKILL.md` and all reference pages.
- CMH-PRINT-01: `SKILL.md` advises exporting to Plain HTML for PDF but does not state that direct print is explicitly not a supported contract.
