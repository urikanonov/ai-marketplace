# commentable-html - development and testing

Maintainer guide for changing the skill itself. For **using** the skill (adding
the layer to an HTML, the per-generation `tools/validate.py` check, the review loop),
see `SKILL.md`. Nothing here runs per generation - it is only for editing the
layer's code and keeping the test suites green.

> **Vendored copies exclude this tooling.** This guide describes the **source-of-record**
> repository, which ships the full dev harness. Trimmed distributions -
> e.g. the agency/playground plugin copy - intentionally exclude the dev-only files
> referenced here (`tests/`, `package.json`, `playwright.config.js`, `node_modules/`). Run
> every command below from a source-of-record checkout, not from a vendored copy.

## Architecture: single source of truth

Edit only the **canonical sources**; everything else is generated.

**Canonical sources (edit these):**

- **`assets/commentable-html.css`** - the layer stylesheet (CSS region body).
- **`assets/commentable-html.js`** - the runtime (JS region body). Its `CMH_VERSION` constant is the single source of the version used across dist filenames, the handshake `<meta>`, and the manifest.
- **`assets/template.shell.html`** - the page shell with `{{CMH_CSS}}` / `{{CMH_JS}}` placeholders, the five-region scaffolding, the toolbar/sidebar UI, and the demo content.

**Generated by `tools/build.py` (never hand-edit):**

- **`TEMPLATE.html`** - the complete, self-contained, runnable inline/standalone template + demo (mermaid, code block, code-review diff, charts-ready). Regenerated from the shell + assets.
- **`dist/`** - the economy bundle: `ECONOMY.html` (the economy starting shell), `commentable-html.v<V>.css` / `.js` / `.assets.js` (the companions copied next to a target), and `manifest.json` (version + SHA-256 per companion). `dist/ECONOMY.html` sits next to its companions so it validates as a self-contained example.

## Build pipeline

```
python tools/build.py            # regenerate TEMPLATE.html + dist/ (css, js, assets.js, manifest.json, ECONOMY.html); prints the inline-vs-economy size delta
python tools/build.py --check    # fail if any generated file is out of sync with the sources
```

After editing the CSS or JS, run `python tools/build.py`, then the suites. `tools/build.py --check` is the drift guard that keeps the inline and economy forms from diverging (wire it into CI). `dist/manifest.json` records the version and a SHA-256 for each companion file.

## Running the tests

Two layers - run both after any change.

### 1. Python suites (no install - just Python 3)

Standard-library `unittest`, no third-party packages. The scripts live in `tools/`
and their tests in `tests/` (alongside the browser specs). From the skill root:

```
python -m unittest discover -s tests -p "test_*.py"
```

(Or `npm run test:py`.) 320 tests total.

| Suite | Covers |
| --- | --- |
| `tests/test_validate.py` (116) | layer checks - every branch mutation-checked, CLI exit codes/flags, CRLF, non-UTF-8, batch, retrofit guards (incl. commented-out root detection for quoted and unquoted attributes, and ignoring comment-like text inside script/style bodies), and an `EconomyTests` class (economy detection, companion refs, version handshake, banner, missing-file, real `dist/ECONOMY.html` positive control). |
| `tests/test_validate_charts.py` (71) | the Chart.js checks, mutation-checked. |
| `tests/test_build.py` (13) | `--check` in sync, generated files match disk, idempotence, inline round-trips from shell+assets, version single-sourced, manifest hashes, registry round-trip, both templates validate clean, the economy-vs-inline size delta, stale dist-file detection, and duplicate-version rejection. |
| `tests/test_mark_handled.py` (25) | append/dedupe/order, unsafe-id rejection, missing block, surgical (only the block changes), CRLF preserved, bundle parsing, CLI. |
| `tests/test_kql_highlight.py` (27) | the offline KQL syntax highlighter - tokenization, dark/light theming, HTML-safety, and idempotence. |
| `tests/test_kusto_link.py` (19) | the "Run in Kusto" deep-link builder - gzip+base64 round-trip, cluster-host validation (incl. rejecting a bare host with a path segment), CLI flags, and deterministic output. |
| `tests/test_inline_images.py` (12) | the `inline_images.py` build tool - local `<img src>` become `data:` URIs (svg/png), remote/data/fragment/absolute/`data-src` sources untouched, `<script>`/`<style>`/`<pre>` regions protected, path-traversal refused, query/fragment handled, missing images reported, `--strict` and `--out` CLI. |
| `tests/test_examples.py` (4) | the worked example (`examples/community-garden.html`) exists, validates with zero warnings, has its images inlined (self-contained), and exercises every content feature. |

### 2. Browser E2E suite (`@playwright/test`, `tests/` + `playwright.config.js`)

**Prerequisites (one-time):**

1. **Node.js 18+** and npm.
2. Install the dev dependencies (Playwright runner + test-only vendored copies of mermaid and a zip reader):
   ```
   npm install
   ```
   If your default npm feed is a locked-down corporate proxy that fails with a JSON/fetch error, use the public registry:
   ```
   npm install --registry https://registry.npmjs.org
   ```
3. Download the Chromium binary Playwright drives:
   ```
   npx playwright install chromium
   ```

**Run:**

```
npx playwright test            # or: npm run test:e2e   - headless, all 196 tests
npx playwright test --ui       # or: npm run test:e2e:ui - interactive browser runner
npx playwright test --headed   # watch it drive a real browser window
npx playwright test 03-comments-text        # one spec file
npx playwright test -g "Export standalone"  # one test by title
npx playwright show-report     # or: npm run test:e2e:report - open the last HTML report
```

**What the 196 tests cover:** load + init (no unexpected console/page errors, light + dark theme, inline vs economy detection); the overflow menu (open/close, Esc, outside-click) + Show/Hide toggle (the floating toolbar hides while the panel is open); the full comment lifecycle - add (synthetic and **real pointer drag**, mouseup and **native right-click**, a **whitespace-only selection is rejected**, a **.cm-skip selection is rejected**, and a **cross-structural selection either anchors exactly or is cleanly rejected with a toast**), the composer **Cancel button**, **empty/whitespace-note rejection**, **composer drag**, edit, delete-one, clear-all, Ctrl+Enter/Esc, multiple simultaneous composers, **cid-mapped click-highlight-to-activate**, the `#hlBubble` on both link-wrapped and **plain** highlights (opens without navigating), and **jump-to-highlight (proven off-screen first)**; code + mermaid comments (mermaid rendered from locally vendored mermaid, node-comment pruning, unhandled-reload restore on the exact same node, **mermaid ring-click activation**, **mermaid card jump**, and the **mermaid Copy-all payload**); the **full Copy-all bundle contract** (header/`Source`/`Where`/`Pinpoint`/occurrence/`In context`/`Offsets`/`Quoted text`/`AGENT INSTRUCTIONS`/exact `HANDLED_IDS_JSON`, fenced code with no prose-only fields, top vs sidebar Copy all, the "No comments to copy" toast, and the **execCommand and prompt() clipboard fallbacks**), and handled-id pruning that keeps the unhandled comment; Save comments (embeds the comment, `<stem>-comments.html`, top and sidebar buttons) / embedded-comments travel / embedded-vs-localStorage merge by `updatedAt` **and disjoint-id union** / Export plain (**unwraps highlights, strips `sidebar-open`, leaves localStorage intact**, `.plain.html`) / Export standalone (**embeds current comments**, reopens showing them, `.standalone.html`) / Export economy (**sidebar button, `.economy.zip`**) / a closing-script-tag injection check; the economy features (Export standalone + Export economy round-trips re-validated with `tools/validate.py`, missing-companion banner, version-handshake banner, mode-gated actions, and Export standalone over HTTP); the **offline guarantee** (a mermaid-free document, with the network-deny route recording every attempt, reaches out to nothing); an **overlapping selection never corrupts the existing comment**, and a comment whose stored text no longer exists **degrades gracefully on reload**; a targeted coverage spec (`11-coverage.spec.js`: duplicate-phrase occurrence anchoring, RTL selection, sidebar empty-state, Escape priority, mode gating); the committed **fixture-freshness guard** (`generate.mjs --check`); and randomized **noise/fuzz** tests that add many comments at random positions (inline elements, entities, emoji, element-boundary ranges with at-least-one-anchored, and a mixed add/edit/delete/clear state machine with a DOM-marks-match-storage invariant) and assert every one anchors to exactly its selected DOM text, is reload-stable (**byte-for-byte** offset round-trip on the raw covered text) with no "Lost anchor" warnings, and prunes when handled. Specs are numbered by area (`01`-`13`); shared fixtures, the offline network-deny, and the mermaid routing live in `tests/helpers.js`. The **diff / code-review layer** is covered by `13-diff.spec.js`: a `pre.cmh-diff` renders into a colored view, the side-by-side / inline toggle switches layout and persists, a diff-line comment highlights structurally and survives the layout toggle and reload, the Copy-all bundle carries the diff anchor + fenced diff line, and the diff interactions add zero network activity.

**Offline / no-CDN by design.** The suite needs **no internet at run time**. Pages open over `file://` (matching how users open these files), and the one feature that normally needs a CDN - mermaid - is served from the locally vendored `node_modules/mermaid` via Playwright route interception (`tests/helpers.js` `routeMermaidLocal`), so the mermaid-node tests run fully offline. A dedicated offline test (`tests/12-features.spec.js`) installs `denyExternalNetwork` (which aborts and **records** every non-local request, including the mermaid CDN) and loads a mermaid-stripped document to prove the layer itself makes zero network calls. Nothing depends on jsdelivr being reachable.

### Sample fixtures (edge cases)

Interaction and fuzz tests run against pregenerated rich fixtures in `tests/fixtures/` (an inline `kitchen-sink.html` and an `economy/kitchen-sink.html` variant whose companion `<link>`/`<script src>` point at the real `../../../dist/` bundle rather than duplicating it), built from the layer plus `tests/fixtures/sample-content.html` - a dense, deliberately hostile document (text split across nested inline elements, adjacent/short text nodes, HTML entities, Unicode + emoji surrogate pairs, RTL runs, links, tables, blockquotes, code blocks). This exercises anchoring far beyond the skill's own demo content.

The fixtures are **generated** and committed. Because they are derived from the current `TEMPLATE.html`/`dist/`, they can go stale when the layer changes.

🔴 **When you add a feature or change the layer or the sample edge cases, refresh the fixtures:** run `python tools/build.py` then `node tests/fixtures/generate.mjs` (`npm run fixtures`). A test (`10-more.spec.js`, `generate.mjs --check` via `npm run fixtures:check`) fails if the committed fixtures are stale, so this cannot be forgotten. To add a new edge case, edit `tests/fixtures/sample-content.html` and regenerate.

## What is and isn't committed

`assets/` (incl. `template.shell.html`), `tools/` (`build.py`, `mark_handled.py`, `validate.py`), `docs/`, the generated `TEMPLATE.html` / `dist/`, `tests/` (the browser specs, the Python `test_*.py` suites, helpers, and the generated `tests/fixtures/`), `playwright.config.js`, and `package.json` are part of the skill. `node_modules/` (Playwright + vendored mermaid + adm-zip), `test-results/`, and `playwright-report/` are generated and gitignored - a fresh checkout re-creates them with the two install commands above.

In a **vendored/trimmed distribution** (e.g. the agency/playground plugin copy) the dev-only
tooling above - `tests/`, `package.json`, `playwright.config.js`, and `node_modules/` - is
**not** shipped; only the runtime, generated outputs, and per-generation tools travel with it.
Run the suites from the source-of-record repo.
