Comment on any HTML,
hand it back to your agent

Turn any standalone HTML into a single-file review surface. Select any paragraph, table cell, code block, KQL query, chart, image, or diagram, leave an inline comment, then copy a bundle straight back to the AI that made it.

Install

Add the marketplace, then install the plugin.

$ 
copilot plugin marketplace add https://github.com/urikanonov/ai-marketplace
copilot plugin install commentable-html@urikan-ai-marketplace

What you get

A full code-review layer baked into one portable HTML file.

Comment on anything

Prose, table cells, code blocks, KQL queries, charts, images, and mermaid diagram nodes are all selectable and commentable.

Single self-contained file

The CSS, UI, and runtime travel inside the document. Email it, archive it, or open it offline - it just works.

Round-trip to the agent

Copy all emits a Markdown bundle plus a machine-readable id list, so the agent acts on your notes and marks them handled.

Export as Portable

Embed the current comments into the file so a review travels with it. Share it and the recipient sees every note immediately.

Handled comments stay gone

Once the agent marks a comment handled, it is pruned on reload. The HTML file is the single source of truth, not the browser cache.

Deterministic tools

Ships stdlib-only Python tools and a strict validator so generated documents are correct by construction, not by luck.

How the review loop works

Feedback goes back to the agent as structured data, not pasted prose.

Generate

The agent produces a commentable HTML report.

Comment

You select text and leave inline notes in the sidebar.

Copy all

Export a Markdown bundle of every open comment.

Reload

The agent acts and marks ids handled; handled comments prune away.

Changelog

Changes to commentable-html. See the full changelog on GitHub.

[1.0.0] - 2026-07-10

Added
  • Offline, single-file commentable HTML review surface: reviewers select any paragraph, table cell, code block, KQL block, chart, image, or mermaid diagram and leave inline comments, then copy or export a bundle back to an agent, with no network dependency.
  • Portable mode (one self-contained file) and NonPortable mode (companion CSS/JS files).
  • Deterministic Python tooling (new_document, inline_images, chart_block, upgrade, validate, and more) plus a standard-library validator that enforces the structural invariants of a generated file.
  • Rich, commentable content: tables, Chart.js charts, mermaid diagrams, KQL blocks, code-review diffs, and inlined images.
  • Each generated file stamps the skill version that produced it in a <meta name="commentable-html-version"> in the head and in the visible footer.
Notes
  • The injected layer is version-agnostic: the region markers, the companion filenames, and the demo storage keys no longer embed a version. The single source of truth for the release version is dev/VERSION; build.py stamps it into the layer constant, plugin.json, the marketplace entry, and each document's version <meta>, and build.py --check guards against drift.
